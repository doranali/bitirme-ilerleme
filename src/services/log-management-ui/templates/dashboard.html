Kayıtlı uç noktalar

                        Graylog · API
                    

                    

                        

                            
                            
                            
                        

                        
Liste boşsa: Operatör rolüyle giriş yapın; kurulumdan sonra Yenile. Hâlâ yoksa Disk eşitle (token kaydından uç satırı yazar). TLS / sertifika özeti için Ayarlar sekmesi.

                        
Kopuk veya hayalet satır: satıra tıklayıp açılan pencerede Listeden kaldır — panel ve nodes.json temizlenir; token kalır (ajan yeniden bağlanırsa satır dönebilir; kalıcı iptal için token iptali).

                        
Panel «Güncel» yalnızca panele gelen heartbeat’i gösterir. 24 saat / 7 gün sütunları Graylog’taki mesaj sayısıdır; «—» ise o sürede eşleşen log yok veya Graylog sorgusu hata verdi (hücre üzerine gelin). Doğrudan doğrulama: Graylog aramasında host:MAKINE_ADI veya source:MAKINE_ADI.

                        
Syslog-only cihazlar (vCenter, firewall) token/aktivasyon yapmadığı için üstteki «Kayıtlı uç noktalar» listesine otomatik satır eklenmez; loglar doğrudan Graylog’a gider. Özet için aşağıdaki Graylog özeti kullanılır.

Syslog / kayıtlı ağ görünürlüğü (Graylog)

                        
Son 7 gün, log_type:network. Önce transport IP (çoğu saha: vCenter); başlığa tıklayınca o IP’den gelen servis (source) dağılımı; servis satırına tıklayınca örnek loglar. source tek fiziksel kutu değildir. İstemci adresi için syslog_sender_ip önceliklidir.

Kurulum ve tokenlar

                        Sihirbaz · gelişmiş
                    

                    

                        
Profil → tek token → tek kurulum bağlantısı (.sh Linux, .ps1 Windows). Fluent Bit ajanı: merkeze UDP 5151 JSON. vCenter / firewall syslog: aynı merkez IP’de UDP veya TCP 1514 (ekstra relay isteğe bağlı). Token için admin.

⚡ Performans (VM Auto-Tune)

                                Compose uyumlu
                            

                            

                                
Öneriler, docker-compose.yml içindeki Graylog/OpenSearch giriş noktası ile aynı formülü kullanır (%20 rezerv; kalan RAM’de Graylog heap %25, OpenSearch %50, min 512 MB). Uygula GRAYLOG_JAVA_HEAP_MB ve OPENSEARCH_JAVA_HEAP_MB yazar; ardından hostta docker compose ile graylog ve opensearch1 otomatik yeniden oluşturulur (Docker soketi gerekir). Kapatmak için ortam: LOG_SYSTEM_HEAP_COMPOSE_RECREATE=0. docker-compose.prod.yml gibi farklı profiller sabit JAVA_OPTS kullanabilir.
